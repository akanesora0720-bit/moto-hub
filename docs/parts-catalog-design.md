# パーツカタログ・検索 設計メモ

## 現状（073 まで）

- `part_listings.manufacturer` / `category` / `compatible_models` は **自由テキスト**
- `/parts` は新着60件のみ（検索なし）
- 表記揺れ（ホンダ / HONDA / ＣＢ４００）で検索が効かない

## ① 最小実装（074 + アプリ）

| 領域 | 内容 |
|------|------|
| DB | `part_manufacturers` / `part_categories`（シードのみ） / `part_models`（学習型） |
| 出品 | メーカー・カテゴリはプルダウン、車種はサジェスト + 「汎用」 |
| 正規化 | 車種・品番は `normalizePartCatalogText()`（NFKC + 大文字化 + 空白除去） |
| 検索 | `/parts?` でメーカー・車種・カテゴリ・キーワード・品番・ASK除外・価格帯 |
| 互換 | 旧テキスト列はトリガーで同期（PDF・既存参照を壊さない） |

**意図的にやらないこと（初期）**

- 数千車種の事前マスタ投入
- 管理画面でのマスタ CRUD
- 全文検索エンジン / `tsvector`
- 車両 `listings` との車種マスタ統合

## ② 将来拡張

- 管理画面でメーカー・カテゴリの追加・並び替え
- `part_models` の統合・重複マージ（CB400SF / CB-400SF）
- 人気車種のシード（トップ50程度）のみ手動投入
- 画像検索、適合表、複数車種対応（中間テーブル）
- 出品料・掲載プラン（流通量に応じた課金）
- 検索ログからのマスタ候補提示
