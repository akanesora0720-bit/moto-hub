# 取引 UI：当事者別の進捗・バッジ・案内

`DEAL_STATUS_LABELS` は**運営・DB上のステータス名**（例: `payout_ready` = 振込準備完了）。  
画面では **`partyDealStatusBadge` / `buyerDealLabel` / `sellerDealLabel` / `partyDealActionHint`** を使い、買い手・売り手で文言を分ける。

## 用語（ユーザー向けに使い分ける）

| 用語 | 意味 |
|------|------|
| **車両代金** | 買い手が売り手の口座へ直接お振込みする金額（税込） |
| **MotoHub手数料** | 売り手宛のプラットフォーム手数料請求書（入金確認後に発行） |
| **取引の完了** | 双方の確認後、運営がシステム上で取引を閉じること（`payout_ready` 以降） |

「成約金振込」は使わない（車両代金の二重振込と誤解されやすいため）。

## 業務フロー（概要）

| 段階 | ステータス | 買い手 | 売り手 | 運営 |
|------|------------|--------|--------|------|
| プレ商談 | inquiry / negotiating / agreed | 商談待ち | 商談待ち | 商談・合意操作 |
| 入金 | awaiting_payment | 車両代金を売り手へ振込 | 入金確認 | 監視 |
| 引渡 | funded | 引取日時登録 | 引渡完了 | 連絡板 |
| 確認 | handover_done / transfer_pending | 完了確認 | 完了確認 | 名変コンプライアンス |
| 完了登録 | payout_ready | 待ちのみ | 取引完了待ち | 取引を完了にする操作 |
| 終了 | completed | 完了 | 完了 | — |

**直接支払:** 車両代金は MotoHub を経由しない。  
**payout_ready:** 双方確認済み。運営が取引を「完了」にする段階（売り手への別途振込を意味しない）。

## `payout_ready` の表示例

| | 進捗（大） | バッジ（小） | 案内 |
|---|------------|--------------|------|
| 買い手 | 完了登録待ち | 確認完了 | ご対応不要。運営が取引を完了にします |
| 売り手 | 取引完了待ち | 完了登録待ち | 車両代金は入金確認済み。運営が取引を完了にします |
